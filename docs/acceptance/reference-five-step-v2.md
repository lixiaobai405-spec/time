## 五步主路径

1. **事务填写**：`POST /api/time-management/intake/check`
   - 校验四栏结构、文本长度、总任务行数和全空输入。
   - 部分类别为空时返回警告，不虚构任务。
2. **AI 拆解确认**：`POST /api/time-management/tasks/decompose` 与 `/tasks/smart-check`
   - 模型拆成稳定 ID 任务；用户可编辑或手动补充。
   - 服务端逐字段检查任务描述、截止时间、可解析工时和轻重缓急。
3. **时间分布诊断**：`POST /api/time-management/distribution/diagnose`
   - 只使用明确小时/分钟工时，按分钟汇总。
   - 最大余数法保证显示占比合计为 100.0%。
   - 返回未参与计算的任务、达标状态、诊断和改进方向。
4. **优先级排序**：`POST /api/time-management/matrix/classify`
   - 按稳定任务 ID 守恒，不增删、不按名称去重。
   - 四象限固定 55/25/15/5，允许空象限。
5. **优化报告**：`POST /api/time-management/report/generate`
   - 同时读取当前任务、时间分布、矩阵和四栏输入。
   - 报告只能引用当前任务，继续执行到期保护、调度冲突和 ID 泄漏校验。

## 持久化边界

- SQLite：用户、凭据哈希、Session 哈希和成功生成的账号报告。
- 浏览器内存：五步草稿、任务编辑、完成勾选、本次会话每日记录。
- 不使用 localStorage 或 sessionStorage 保存账号凭据、草稿或业务正文。
- 刷新或退出会清空浏览器内存中的五步草稿和每日记录。
