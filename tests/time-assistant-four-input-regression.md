# 时间管理助手 · 四栏输入回归测试内容

> 用途：将下面四段虚构内容分别复制到时间管理助手的“昨天、今天、明天、后天”输入框，用于测试“AI 检查 → 任务提取 → 矩阵判定 → 优先级报告”。
>
> 内容不包含真实客户信息、账号、密码、API key、token 或其他敏感信息。

## 昨天 · 目标复盘

```text
目标：昨天18:00前完成季度项目复盘报告，并与团队确认下阶段改进方向。

结果：复盘报告已经完成并发送给团队，共整理出3个主要问题和5项改进措施，整体完成度约90%。

差距：原计划同步确定每项措施的负责人，但目前还有2项措施没有明确负责人。

原因：下午临时增加了一次跨部门会议，压缩了团队讨论时间；同时会前没有提前收集负责人意向。

改进：今天上午先向团队收集负责人意向，并在今天11:30前完成2项措施的责任人确认。以后召开复盘会前一天发送议题和待确认事项。
```

## 今天 · 关键工作

```text
1. 今天11:30前确认复盘报告中剩余2项改进措施的负责人，并在团队任务表中完成登记。

2. 今天15:00前审核A项目第一版实施方案，重点检查目标、进度、风险和验收标准；输出不少于3条具体修改意见。

3. 今天17:00前与产品和技术负责人召开30分钟项目风险会议，确认本周最重要的3项风险、负责人和处理期限。

4. 今天18:00前整理会议结论并发送团队，确保每项行动都有负责人、截止时间和可验证的完成标准。
```

## 明天 · 短期目标

```text
明天17:00前完成A项目第二版实施方案。

方案需要包含项目目标、任务分工、关键里程碑、风险清单和验收标准；至少明确5项主要任务，每项任务都要标注负责人和截止时间。

完成后邀请产品负责人和技术负责人评审，收集反馈并形成一份修改清单。验收标准是方案内容完整、两位负责人均已确认、所有评审意见都有处理结论。
```

## 后天 · 中长期目标

```text
未来90天内建立一套可执行的团队项目管理机制，使重点项目按期完成率从当前约70%提升到85%以上。

第一个月完成统一的项目计划、周报和风险清单模板，并选择A项目试运行。

第二个月根据试运行结果优化流程，确保所有重点任务都有负责人、截止时间和验收标准。

第三个月推广到全部重点项目，每周检查一次项目进度，每月进行一次复盘。

最终验收标准：重点项目模板使用率达到100%，风险问题在发现后24小时内完成登记，项目按期完成率达到85%以上。
```

## 预期现象

- [ ] AI 检查应倾向于通过。
- [ ] “昨天已经完成复盘报告”不应被重复生成待办。
- [ ] “确认剩余2项负责人”应成为今天的任务。

> 模型输出存在一定随机性；若 AI 检查未通过，请记录对应输入框下的 `issue` 和 `suggestion`，不要通过反复点击绕过检查。
