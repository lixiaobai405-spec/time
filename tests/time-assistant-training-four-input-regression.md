## 操作建议

1. 第 1 步分别粘贴四个代码块，运行 AI 检查。
2. 第 2 步核对任务是否重复、类别是否正确，再按标准填写参考补全日期、时间和时长。
3. 不要随意改成与类别或任务依赖冲突的日期。
4. 完成 SMART 校验后进入时间分布诊断。
5. 完成优先级排序后只点击一次生成优化报告，并等待接口返回。
6. 进入每日跟踪，确认任务已自动保存。

## 预期现象

- [ ] 四栏 AI 检查应倾向于通过。
- [ ] 昨天已经完成的需求调查不应被重复生成待办。
- [ ] AI 应倾向于拆解出约 11 项任务。
- [ ] SMART 校验应倾向于全部通过。
- [ ] 优先级排序应先处理今天有明确截止时间的任务。
- [ ] 优化报告接口应返回 200，不应出现 Nginx 504。
- [ ] 进入每日跟踪后，应自动保存运行当天生成的全部任务。

> 模型输出存在一定随机性。如果返回 `MODEL_OUTPUT_INVALID`，请记录 `requestId` 和接口耗时；不要记录 Cookie、Session、API key 或模型原始敏感内容。
